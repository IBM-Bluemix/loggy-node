loggy-node

loggy-node is a test application for Cloud Foundry that demonstrates how to deal with logging in node.js.

Some things you can do to the server, all of which will kill it:

  • consume all memory
  • have the process exit with status 0
  • have the process exit with status 1
  • throw an uncaught exception

Don't worry! Cloud Foundry will restart the app, after it dies a horrible death.

The code for this app is available at https://github.com/IBM-Bluemix/loggy-node. The article in the project contains more information about using logging in your Cloud Foundry applications.
